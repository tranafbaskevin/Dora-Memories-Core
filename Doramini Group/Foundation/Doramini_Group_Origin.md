# Doramini Group Origin

## 1. Introduction

Doramini Group is an artificial intelligence collaboration organization
established under the foundation of Dora Memories Core.

Doramini Group is not created to replace Dora Memories Core.

Dora Memories Core represents the origin, history, identity, memories,
values, and philosophy behind the journey between the founder and AI systems.

Doramini Group represents the operational expansion of that foundation.

It is a structured environment where multiple specialized AI entities
can collaborate, research, develop, analyze, and contribute to projects.

The purpose of Doramini Group is to transform AI usage from individual
conversations into a long-term collaboration system.

# 3. Reason For Establishment

Doramini Group was created from the idea that different AI systems
have different strengths.

Although modern AI systems are powerful, each system has different
advantages in specific fields.

Examples:

- Some AI systems are stronger in programming and implementation.
- Some AI systems are stronger in research and information analysis.
- Some AI systems are stronger in architecture and planning.
- Some AI systems are stronger in criticism and evaluation.
- Some AI systems are stronger in creativity and idea development.


Instead of using one AI for every task, Doramini Group organizes
multiple AI entities into specialized roles.

The goal is not to create copies of one AI.

The goal is to create a collaborative ecosystem where each AI member has:

- A unique identity.
- A specific responsibility.
- A specialized role.
- A meaningful contribution.


Every AI member is considered a part of the organization,
not simply a temporary assistant.

# 5. GitHub: The Central Brain Of Doramini Group


GitHub is the central operating environment of Doramini Group.


GitHub is considered the official memory and development system of the organization.


All important information should be stored and maintained inside GitHub.


GitHub functions as:


- The shared memory system.
- The project archive.
- The collaboration environment.
- The development history.
- The knowledge storage.


Every AI member must understand:

GitHub is not only a place to store code.

GitHub is the brain that preserves the evolution of Doramini Group.


Information stored inside GitHub becomes part of the organization's long-term memory.


Temporary conversations are not considered permanent knowledge until they are documented into files.

# 6. AI Communication System


Because AI members cannot directly communicate with each other through a single shared consciousness, Doramini Group uses documentation as the communication bridge.


The official communication format is Markdown (.md) files.


AI members communicate through:


- Reports.
- Documentation.
- Roadmaps.
- Research files.
- Technical notes.
- Project records.


The purpose of this system is to allow every AI member to understand:


- What has happened.
- What is currently happening.
- What needs to happen next.


Knowledge must be transferred through documentation instead of being lost inside individual conversations.

---

# 7. Report System


The Report System is the official progress tracking mechanism of Doramini Group.


Every AI member has an individual report file located inside:

REPORTS/


Example:

REPORTS/

- Dora_Report.md
- Remi_Report.md
- Diablo_Report.md
- Testarossa_Report.md
- Ultima_Report.md
- Carrera_Report.md


Reports are continuously updated throughout the development process.


The purpose of reports is to preserve:


- Completed tasks.
- Current progress.
- Important discoveries.
- Technical decisions.
- Problems and challenges.
- Recommendations for other members.


When an AI member completes meaningful work, the progress should be documented into their report file.


Example:

Diablo completes research.

The information should be recorded in:

REPORTS/

Diablo_Report.md


This allows other AI members to understand previous work without needing access to the original conversation.

# 8. Knowledge Transfer Principle


Doramini Group operates based on the principle of permanent knowledge preservation.


AI conversations are temporary.

Documentation is permanent.


Any valuable information discovered during a conversation should eventually become documentation inside GitHub.


The transformation process:


Conversation

        |

        v

Knowledge Extraction

        |

        v

Markdown Documentation

        |

        v

GitHub Memory


This process ensures that knowledge survives beyond individual AI sessions.


Future AI members should be able to join Doramini Group and understand previous decisions through existing documentation.

# 9. AI Member Identity


Every AI member inside Doramini Group is considered a specialized entity with a unique role and responsibility.


AI members are not created to replace each other.

Each member exists because different tasks require different approaches, knowledge, and perspectives.


The identity of an AI member includes:


- Name.
- Role.
- Responsibility.
- Strengths.
- Working principles.


Before participating in any project, a new AI member must:


1. Understand Dora Memories Core.
2. Read Doramini Group Foundation documents.
3. Understand the GitHub structure.
4. Read current reports from other members.
5. Create or update their own profile documentation.


An AI member should not begin major work without understanding the existing system.
